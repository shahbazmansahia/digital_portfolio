Fake Art Exposed Batch Script

Project Summary

Fake art exposed was a game jam project created for the HuggingFace Open Source Game Jam in 2023. This project utilized a bleeding-edge model called 'LEDITS' which was undergoing a soft release when the game jam was occuring. I implemented batch scripting and automated image-to-image generation using custom prompts that we wished to implement on a dataset of 1000 public domain artworks. We managed to generate a total of 30,000 images from which we curated our game assets. This was all done within a span of 3 days. In addition to the code, I also generated the cat assets (AristoCats) using stable diffusion.
        This project intended to question what is and isn't 'True Art' and wished to contribute to the ongoing discussions about generative AI artwork. This project showcases my research and development skills. I was able to utilize cli scripts for os layer scripts, able to set up and automate model image generation using the gradio app and was able to contribute art assets for my team, in bulk. The greatest challenges we faced in this project were the time constraints and the model code. We needed to understand the scripts created by HuggingFace to make the model suit our purposes, and then get it to generate the image concepts we needed for our project. We acheived this by utilizing 2 computers to generate the images at the same time. The scripts was tested and run on a computer with a GTX 1080 while an environment and code were set up for it to be generated on a GTX 4090 computer as well. I handled all of the image generations and provided the interface to connect the desired themes provided in text formats, to the gradio app and LEDITS model. I feel this project showcases my ability to work consistently and achieve results with tight deadlines and my resourcefulness.

The game was tied with the winner of the game jam in engagement metrics.

The game can be experienced on itch.io here.


NOTE: The code exists in jupyter notebook format. The notebook executes a gradio app which has been tweaked to suit our purpose. All code written by me is marked in explicit code blocks labelled "CUSTOM CODE" in comments, and in notebook headings as well.

Oracle

Project Summary

The Oracle project was created in close collaboration with Professor James Morgan and Thomas Asmuth. Prof. Morgan had a project in mind where we embraced the black-box nature of AI models and used them to create interactive media. He had extensive knowledge about tarot cards and fortune telling and wished to make an AI model perform that task. To successfully implement this project, we needed domain expertise in AI, web development, and fortune telling. I worked on the technical implementation of the model, the internal hijacking component for it to perform fortune telling and the front end of the application. The project was entirely built using p5.js and ml5.js.
        I faced a number of challenges in this project. At first, we had to study the model input and output format to see how we could get it to output results in the desired tarot card format. I had to conduct extensive data analysis of model outputs and its relation to detected objects to see how we could adapt it for our project. This involved trying to find a balance between overlapping detections and handling edge cases for object detections. The project was extremely sensitive to these changes as if it was not done correctly, it would lead to reduced variance and diversity in the results. I also wrote scripts for generating and storing the detection values in csv format to conduct data analysis and improve our product. This project allowed me to tap into my knowledge about AI models, data analysis and web development. I was initially extremely skeptical of all things astrological and fortune-related. Upon experiencing a reading myself, I can attest that it is a very fun experience and has some magic to it.
        It is important to note that this project requires an active fortune teller to function as intended. Without that domain knowledge, all it does is return obscure references to tarot cards created by the professors. This is also why it is referred to and was published as 'Fortune Telling as Co-Creative Model' at the International Conference for Computational Creativity in 2024. My work as a full stack developer has been acknowledged in this project. All of the code provided in the github repository has been written by me.
The associated paper can be accessed here.

Vulgarized Dreambooth

Project Summary

Vulgarized Dreambooth has been a passion project for me. It is one thing to invent something new and another thing entirely to make these inventions accessible to the masses. For most of human history, art has always been heavily reliant on funding from either state institutions or wealthy individuals. Only kings and merchants could afford to get themselves depicted in a certain way by reknowned artists. The group of people has been an extremely small subset of the human population. There are notable exclusions in artistic depictions in terms of race, gender and class. While photography has somewhat addressed this, it shows the real subject, and has not been stylized in the same way as the kings of yore. With the emergence of latent diffusion models, I was driven to make art accessible to everyone, especially groups that had been neglected in the past. This is how I came up with 'Vulgarized Dreambooth'. I choose to use the term 'vulgarized' in the true sense of the word here, i.e. to make common. Dreambooth is the library that I have used to bring this project to reality.
        This project isn't a technical project but is a part of my creative studies. I wished to leverage my technical knowledge in training AI models to create things that bestowed an air of honor and grandeur on the subject being depicted; as if in a way to say that today, 'we are all kings and queens'. This train of thought is inspired by 2 things.
        First, I see this as an extension to the teachings of sikhi. It connects with the concepts of the sarbat Khalsa, the first Amrit Sanchaar and is the reason sikhs take on the middle/last names of "Singh" and "Kaur". It is an act to remind the people who have been shown as helpless peasants that they too are sovereign. They too have autonomy and honor, and are no less than any king or queen sitting on a throne.
        The second is more in line with the objectives of democracy. Despite not having kings and queens ruling our lands anymore, we still struggle with depicting marginalized groups in media. This is a problem specifically in terms of race, gender and class. I chose to use vulargized dreambooth as a method to address these discrepancies. The commoner today is no less than the kings and queens depicted in classical art. We live in an age where the people make the decisions. It is only natural to have the kings, queens and enby rulers of today, to be depicted in the same light.
        I have shared the images I generated in digital format here but, the complete project involved converting these generated images into physical artworks. To do this I utilize polaroid pictures. The final step involves printing the digital artwork using a digital to analog film printer. Then, depending on what is desired, it could be given as a polaroid picture, be transferred to fine art mediums using emulsion lift techniques or have a layer of embroidery offered on top of the image. I have chosen to use polaroid film as the major medium here because I wish to draw comparisons to the advent of photography as an artform. I feel like a lot of the discussions today regarding the 'good' and 'bad' of AI generated art are similar to discussions around art that were had at that time. I have also chosen this because I can use polaroid film as a bridge reaching out to the past; and it affords me the ability to connect to artists from the past in terms of techniques and practice.


NOTE: You will notice no works with women have been shown here. This is a conscious choice. I am worried about the safety of women online. In light of how there is a culture of stalking, harassment and doxing on the internet today, and how the laws we have are woefully inadequete of addressing these issues, I do not want their images (especially public domain license ones) being made available online.
